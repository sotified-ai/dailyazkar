# SECTION 1 — Rendering & Crawlability Verification

## Status: ❓ Needs Confirmation

Your earlier crawl indicated near-zero extractable HTML text.

If content is client-rendered only, this is a ranking bottleneck.

Google prefers server-side or static rendering for JS-heavy sites.
Reference:
[https://developers.google.com/search/docs/crawling-indexing/javascript/dynamic-rendering](https://developers.google.com/search/docs/crawling-indexing/javascript/dynamic-rendering)

---

## PLAN 1 — Rendering Hardening

### Objective

Ensure Morning & Evening pillar pages are fully readable in raw HTML source.

### Tasks

1. Add pre-render or static generation for:

   * `/morning-azkar`
   * `/evening-azkar`
   * Homepage

2. CI check:

   * Fail build if rendered HTML < 800 words on pillar routes

3. Validate via:

   * View Source
   * GSC URL Inspection (Rendered HTML view)

### Deliverables

* `prerender/routes.json`
* `prerender/report.json`
* CI gate rule

### Acceptance Criteria

* Full dua content visible in initial HTML
* No dependency on runtime JS for primary text
